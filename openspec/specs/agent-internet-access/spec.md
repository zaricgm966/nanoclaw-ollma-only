# Agent 联网能力规格

## 状态
草案

## 概述

NanoClaw 的 Ollama-only agent 可以在需要最新外部信息时，通过内建联网工具执行搜索和网页抓取，再基于抓取结果生成答复。

## 需求

### 需求：联网搜索工具
NanoClaw 必须提供可由 agent 调用的联网搜索工具。

#### 场景：搜索最新信息
- 假设用户询问需要最新信息的问题
- 当 agent 判断需要联网时
- 那么系统会执行搜索并把结果返回给 agent

### 需求：网页抓取工具
NanoClaw 必须提供可由 agent 调用的网页抓取工具。

#### 场景：读取搜索结果页
- 假设 agent 已拿到一个网页链接
- 当 agent 需要进一步确认网页内容时
- 那么系统会打开目标网页并抽取可读文本返回给 agent

### 需求：自动兜底
对于明显需要联网的问题，NanoClaw 必须在模型未主动调用工具时提供自动兜底。

#### 场景：模型首轮未调工具
- 假设用户问题明显需要联网
- 且模型首轮回答没有调用工具
- 当 runner 检测到这种情况时
- 那么系统会自动执行一次搜索并把结果回灌给模型继续生成回复
