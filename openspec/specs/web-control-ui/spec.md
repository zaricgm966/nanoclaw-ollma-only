# Web 控制台规格

## 状态
草案

## 概述

NanoClaw 可以提供一个可选的本地 Web 控制台，用于查看运行状态、会话、群组、日志、任务、运行配置，并直接在浏览器中与本地 agent 对话。该控制台是现有消息驱动系统之外的一层可视化界面，不替代消息渠道本身。

## 需求

### 需求：可选的 UI 层
NanoClaw 必须支持一个可选的 Web 控制台，但不能替代现有的消息驱动工作模式。

#### 场景：UI 未启用
- 假设 Web UI 未配置或未启用
- 当 NanoClaw 运行时
- 那么渠道、调度与 agent 执行仍然正常工作

#### 场景：UI 已启用
- 假设 Web UI 已启用
- 当用户打开本地浏览器界面
- 那么用户可以查看 NanoClaw 暴露出的运行状态信息

### 需求：运行态可见性
Web UI 必须提供系统状态、渠道、群组、日志、任务与运行配置的可见性。

#### 场景：加载仪表盘
- 假设 NanoClaw 正在运行
- 当仪表盘页面加载时
- 那么用户可以看到服务健康状态、已连接渠道以及最近活动

#### 场景：查看运行配置
- 假设 NanoClaw 已启用 Web UI
- 当用户打开设置页面时
- 那么界面会显示当前 Ollama、代理、容器与 Web UI 的只读配置快照

### 需求：群组与会话可见性
Web UI 必须允许查看已注册群组以及最近消息。

#### 场景：查看某个已注册群组
- 假设某个群组已经注册
- 当用户打开该群组详情页时
- 那么界面会显示最近消息与可用的会话元数据

#### 场景：查看最近会话
- 假设数据库中已有聊天记录
- 当用户打开会话页面时
- 那么界面会显示最近活跃聊天以及其最近消息窗口

### 需求：直接对话
Web UI 必须支持通过本地浏览器直接向 NanoClaw 发送消息，并收到 agent 回复。

#### 场景：浏览器内直接聊天
- 假设 Web UI 已启用且本地 agent 可用
- 当用户在直接对话页输入一条消息并发送时
- 那么系统会调用当前本地 agent 并在页面中显示回复

#### 场景：直接对话会话独立保存
- 假设用户已经通过浏览器发送过多条消息
- 当用户刷新页面后再次进入直接对话页时
- 那么界面仍能读取这条本地 Web 会话的历史记录

#### 场景：消息变化时自动滚动
- 假设直接对话页正在显示消息列表
- 当用户发送新消息或 assistant 产生新回复时
- 那么界面会自动滚动到聊天区域底部

#### 场景：携带客户端环境信息
- 假设用户从浏览器发起直接对话请求
- 当请求被送到 NanoClaw 后端时
- 那么请求中会携带浏览器 `userAgent`，供 agent 判断当前客户端系统环境

#### 场景：渲染 Markdown 回复
- 假设 assistant 回复包含 Markdown 或 GFM 内容
- 当消息显示在直接对话页时
- 那么界面会按 Markdown 规则渲染段落、代码块、列表和表格等内容

#### 场景：逐段显示回复
- 假设 assistant 正在生成一条较长的回复
- 当浏览器正在接收流式响应时
- 那么界面会逐段显示正在生成的内容，而不是只在结束后一次性显示完整消息

#### 场景：展示 thinking 阶段
- 假设 assistant 在生成正式回答前产生了 thinking 内容
- 当浏览器正在接收这条流式回复时
- 那么界面会把 thinking 和正式回答分开展示

#### 场景：折叠已完成的 thinking
- 假设某条 assistant 消息已经包含完整的 thinking 内容
- 当用户在直接对话页查看这条消息时
- 那么用户可以展开或折叠该 thinking 面板，而不影响正文阅读

### 需求：日志查看
Web UI 必须支持应用日志和按群组划分的执行日志查看。

#### 场景：查看最近日志
- 假设 NanoClaw 已经产生日志
- 当用户打开日志页面时
- 那么界面会显示应用日志以及最近的容器日志条目

### 需求：本地静态托管
NanoClaw 主进程必须能够直接托管 Web UI 的构建产物，而不是依赖开发服务器。

#### 场景：直接访问控制台端口
- 假设前端已经构建完成且 Web UI 已启用
- 当用户访问 Web API 监听端口下的根路径或 SPA 子路由时
- 那么主进程会返回前端入口页面与对应静态资源
