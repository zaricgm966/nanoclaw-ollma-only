# 变更管理规格

## 状态

草案

## 概述

NanoClaw 使用 OpenSpec 记录重要的功能、架构与流程变更。

## 需求

### 需求：重要变更必须先有提案

所有重要改动都必须先创建一个 OpenSpec change 目录，并包含 proposal、design 和 tasks 文档。

#### 场景：开始一个重要改动

- 假设贡献者准备增加一个重大功能或架构变更
- 当其开始规划工作时
- 那么他需要先在 `openspec/changes/<change-id>/` 下创建对应目录

### 需求：任务追踪可检查

每一个 OpenSpec change 都必须包含可以持续更新的任务清单。

#### 场景：检查实现进度

- 假设某个 change 正在进行中
- 当贡献者打开 `tasks.md`
- 那么他可以看到尚未完成与已经完成的实现步骤
