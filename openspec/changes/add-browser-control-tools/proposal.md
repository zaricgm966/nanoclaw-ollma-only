# 提案：增加 agent 浏览器控制工具层

## 背景

当前 NanoClaw 已经具备基于 Puppeteer 的 `web_search` 和 `web_fetch` 能力，但 agent 仍无法围绕同一个浏览器页面完成多步操作，例如：

- 打开指定网页
- 读取页面结构快照
- 点击页面中的按钮或链接
- 向输入框填写内容
- 在同一页面中回退、前进、滚动与截图

这使得现有浏览器能力更偏向“检索”和“抓取”，还不足以支撑类似 PinchTab 的浏览器操作体验。

## 目标

- 为 Ollama-only agent 增加一组可直接调用的浏览器控制工具
- 复用现有 Puppeteer 单例和浏览器启动配置
- 支持基于页面快照的元素定位与后续点击、输入、选择等操作
- 让 agent 可以围绕“当前活动页面”执行连续浏览器任务

## 非目标

- 不在这一轮增加 HTTP server 封装
- 不在这一轮增加多标签页管理
- 不在这一轮增加登录态持久化策略
- 不在这一轮增加浏览器扩展依赖
