# 变更提案：修复 open_app 串台并优化聊天图片展示

## 背景

当前 Web 直聊已经支持宿主机 `open_app` 与 `take_screenshot`，但在某些上下文下，`open_app` 的执行结果会被上一轮截图历史污染，最终回复成截图成功。与此同时，聊天中的图片仍然使用普通 Markdown `img` 标签渲染，放大查看体验较弱。

## 目标

- 修复 `open_app` 在 Web 直聊中的错误回复
- 过滤与当前宿主机动作能力冲突的旧 assistant 历史
- 将聊天中的图片切换为 `antd` 的 `Image` 组件，支持点击放大

## 非目标

- 不在本次变更中引入完整媒体库管理
- 不在本次变更中修改 Telegram 等渠道的图片展示方式
