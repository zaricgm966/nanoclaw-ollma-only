# 设计说明：open_app 与聊天图片修复

## open_app 修复策略

对于明显的宿主机动作请求，如果模型未主动发出工具调用，runner 不再把宿主机结果重新交给模型自由发挥，而是直接生成确定性的成功或失败回复。这样可以避免历史上下文把 `open_app` 错答成截图。

同时，Web 直聊在构造 prompt 时会过滤旧的“无法打开本地软件”与类似限制性回答，避免这些过期内容继续污染当前会话。

## 图片展示策略

聊天中的 Markdown 图片通过 `react-markdown` 自定义组件改为 `antd` 的 `Image` 组件渲染。这样保留 Markdown 能力，同时获得点击预览与放大体验。
